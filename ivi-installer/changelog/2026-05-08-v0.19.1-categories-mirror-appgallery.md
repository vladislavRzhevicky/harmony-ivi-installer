# 0.19.1 - Category filter now mirrors AppGallery's tab tree

The Category dropdown used to show our own 9-bucket taxonomy
(audio / video / navigation / messaging / auto / keyboards /
browsers / tools / other). It didn't match what users see in
AppGallery itself, and it relied on a brittle Russian-`kindName` →
our-bucket map that occasionally guessed wrong (e.g. tagging a music
streaming app as `video` because its `kindName` was "Развлечения").

Categories now mirror the real AppGallery "Categories → Apps" sub-tab
tree - the same labels under which the user finds apps inside the
AppGallery web app:

```
cars · navigation · travel · entertainment · photo & video ·
social · news & reading · tools · lifestyle · shopping · top · other
```

Each catalog entry's category is set to the URI-section it appeared
under during the live fetch, not derived from the per-item `kindName`.
``DEFAULT_CATEGORY_URIS`` orders specific buckets BEFORE ``top`` so an
app that appears in both Cars and Top gets tagged with `cars` (the
specific bucket); items that only surface in Top - AppGallery's
editorial featured list - keep `top` as their fallback tag.

## Code

- `catalog.py` - `CATEGORIES` rewritten; `_APPGALLERY_KIND_MAP` and
  `_appgallery_category` removed; `_appgallery_item_to_entry` takes
  the section label and emits it directly.
- `sources/appgallery_index.py` - `DEFAULT_CATEGORY_URIS` reordered
  so `top` is last (specific-bucket-wins on dedup).
- `ui/store_tab.py` - `CATEGORY_DISPLAY` rewritten with all 12 new
  labels.
- `resources/extras.json` and `scripts/build_extras.py` -
  re-bucketed the 40 curated entries: audio/video → entertainment,
  auto → cars, messaging → social, browsers → tools, navigation
  unchanged, tools unchanged.

## Tests

`test_catalog.py` and `test_catalog_store.py` rewritten to use the
new category strings; AppGallery-conversion tests now exercise the
URI-section-as-category invariant. 299 passed.
