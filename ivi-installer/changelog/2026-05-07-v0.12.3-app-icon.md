# 0.12.3 - App icon

The bundle now ships with a custom icon (`icon.png` at the repo root,
1254×1254 - IVI dashboard motif with an upload-to-package mark and a
green-check status pip).

## Pipeline changes

Briefcase 0.4.x for the macOS-app target expects a pre-built `.icns`
and won't auto-convert from PNG. Two new pieces:

- `scripts/make_icns.sh` - wraps `sips` + `iconutil` to bake an `.icns`
  with all the standard Apple sizes (16/32/64/128/256/512/1024 + @2x).
  Idempotent. Sanity-checks that the source is square and ≥512 px.
- `scripts/build.sh` - runs `make_icns.sh` as step 0 (before
  `briefcase update`), and now passes `--update-resources` to
  `briefcase update` so a swapped-out icon actually flows into the
  bundle on subsequent builds (without it briefcase silently keeps
  the previous icon).

## Pyproject

- `[tool.briefcase.app.ivi_installer]` gained `icon = "icon"`.
  Briefcase resolves this to `icon.icns` next to the project root.

## Gitignore

- `icon.icns` and `icon.iconset/` excluded - they're regenerated from
  `icon.png` on every build. Only the source PNG is checked in.
