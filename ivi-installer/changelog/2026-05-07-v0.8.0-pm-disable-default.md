# v0.8.0 - pm-disable becomes the default install strategy

The strategy switcher's default flips: `pm_disable_install` is now the
primary path; `hdb_broker_install` is the fallback. The labels in
`strategies.py` already framed pm-disable as "primary" - the runtime
default now matches.

## Why

- The competitor-derived `pm_disable_install` doesn't need the on-device
  broker daemon to be running, so it works out of the box on a fresh
  car without the redeploy-broker step.
- Lighter and faster than routing through the HDB broker socket.
- HDB broker stays available as the explicit fallback for firmwares
  where pm-disable misbehaves.

## Where the default lives

- `ivi_installer/strategies.py` → `cascade_order()` falls back to
  `"pm_disable_install"` when `primary` is `None`.
- `ivi_installer/ui/main_window.py` → `DEFAULT_PRIMARY_STRATEGY =
  "pm_disable_install"`. Persists per-user via `settings`, so anyone
  who already toggled to HDB keeps that choice.
- Tests in `tests/test_strategies.py`, `tests/test_main_window.py`,
  `tests/test_installer.py` updated to assert the new default.

## Compatibility

- No API changes. `install_cascade(..., primary_strategy=...)` still
  accepts either name; only the default for `primary_strategy=None`
  changed.
- Users with a previously-saved `primary_install_strategy` setting
  keep their selection.
