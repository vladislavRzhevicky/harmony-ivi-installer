# 0.12.2 - Grant runtime perms picker now sees apps on screen users

The Tools tab dropdown was missing apps that the user had just
installed. Reported case: installed "Start", switched to Tools tab,
saw only "Radio" - no Start, no other side-loaded apps.

## Root cause

`ThirdPartyPackagesWorker` ran `pm list packages -3` without
`--user`. The shell user (uid 2000) maps to **user 0**, which on
Huawei automotive heads is HEADLESS - no screens, no app bindings.
User-installed apps live under screen users (10/11/12/13). So the
picker was effectively asking "what's installed for the headless
system user?" and getting roughly nothing back. "Radio" leaked through
because it's a factory pre-install with a binding to user 0 too.

This is the same shape of bug that `_strategy_pm_disable_install` had
before v0.12.0: assuming user 0 was the right context on Huawei IVI
when in fact it's the wrong one.

## Fix

`ivi_installer/ui/workers.py` - `ThirdPartyPackagesWorker` now:

1. Runs `pm list users` once via `devices.list_android_users(serial)`.
2. Picks running non-zero user ids (typically 10, 11, 12, 13 on Deepal).
3. Falls back to `[10, 11, 12, 13]` if the probe came back empty.
4. Runs `pm list packages -3 --user N` for each, unions the results,
   sorts, and emits.
5. Reports an error only if every per-user probe failed (so a single
   stuck user doesn't suppress the whole picker).

Result: anything you've installed via this tool - no matter which
screens it landed on - shows up in the picker.
