# 0.15.0 - Store sections + sort

The Store tab grew two browsing affordances on top of the live F-Droid
catalog:

## Sections (chip row above the toolbar)

Mutually-exclusive chips, one always active:

- **All** - every entry the catalog ships (default).
- **New** - packages whose `metadata.added` is within the last
  `SECTION_RECENCY_DAYS` (30) days. Driven by `CatalogEntry.added_at`,
  populated from F-Droid v2 `metadata.added` (ms epoch).
- **Recently updated** - same window, but on `last_updated_at`
  (`metadata.lastUpdated`).
- **Tested** - entries flagged `tested=True`. Replaces the old
  "Tested only" checkbox in the toolbar.

The "Tested only" checkbox is gone; its job is now the **Tested**
chip.

## Sort dropdown (toolbar)

A new pill next to **Source**:

- `Name (A-Z)` (default)
- `Recently updated` - `last_updated_at` desc, name as tiebreaker
- `Recently added` - `added_at` desc
- `Size (small → large)` - `size_mb` asc; entries with no size sort
  to the end.

Sort is applied AFTER all filters (search × category × source ×
section), so a section preset combines naturally with a sort key.

## What F-Droid does NOT give us

A real "Top by popularity" can't be built from the F-Droid index -
F-Droid intentionally publishes no download counters and no rating
data. Aurora and IzzyOnDroid don't fill that gap either. So instead
of faking a Top, the chip row centres on **time-based discovery**
(New / Updated) plus the curated **Tested** signal, which are the
real signals available.

## Code

- `catalog.py` - `CatalogEntry.added_at` / `.last_updated_at`,
  populated by `from_fdroid_index`. New `SECTIONS`,
  `SECTION_RECENCY_DAYS`, `SORT_OPTIONS`, `sort_apps`. `search()`
  picked up `section` and `now_ms` kwargs (the latter injectable for
  deterministic tests).
- `ui/store_tab.py` - `_section` / `_sort_key` state on the tab,
  chip strip with active-pill styling, `Sort` combo, removed
  `tested_check`. `_apply_filters` runs `search` then `sort_apps`.

active-state styling, sort dropdown reorders `_filtered_ids`.
298 passed total.
