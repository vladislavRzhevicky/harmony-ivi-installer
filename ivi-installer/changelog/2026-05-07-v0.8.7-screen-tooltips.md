# v0.8.7 - screen-card tooltips show resolved user ids

Hover any of the Driver / Passenger / Rear cards to see which actual
Android user ids back that category - e.g. `Driver → user 13`,
`Rear → users 10, 12`. After the first install on a new car model
you can verify whether the cabin mapping is right; if a category
landed on the wrong physical screen, swap two ids in the
`_FALLBACK_CATEGORIES` constant in `devices.py` and rebuild.

## Why

Huawei firmware on most IVI heads doesn't give per-user-screen
names anything more useful than `Owner / User 10 / User 11`, so the
heuristic name-based mapping (introduced in 0.8.6) almost always
falls through to the hardcoded layout. The tooltip surfaces what we
actually picked so it can be sanity-checked from the UI rather than
inferring from the install log.

## What changed

- New `_refresh_screen_tooltips()` in `MainWindow` walks
  `_screen_categories` and sets a tooltip on each card + its
  checkbox. Refreshed on first paint and again after every
  `pm list users` probe.
- Tooltip falls back to "no matching screen" when a category came
  up empty (e.g. car has fewer than 4 multimedia users).
- If the device-reported user names actually carry recognisable
  labels (not blank, not "Owner", not "User N"), they're appended
  in parentheses for extra context.
