# 0.21.0 - Drop .xapk ingestion

`.xapk` archives never actually installed on Deepal/Avatr. The
extraction code worked (we read `manifest.json`, gathered splits and
OBB), but both bypass strategies refused multi-APK input because
Huawei's multi-APK bridge command hardcodes the Avatr-only installer
package - and OBB files were extracted but never pushed. The UI showed
a `XapkWarningBanner` saying "this doesn't work" while still accepting
the drop. Conflicting story, dead code path.

This release removes `.xapk` end-to-end so the input boundary matches
what the bypasses can actually do.

## Removed

- `XapkWarningBanner` widget + `_AlertGlyph` (its only consumer).
- `obb_files` field on `InstallContext` - no strategy ever pushed OBBs.
- `_resolve_xapk_contents` / `_resolve_xapk_obb` helpers.
- `tempfile.TemporaryDirectory` "holder" in `build_context_from_path`'s
  return - function now returns a single `InstallContext`. Both
  `install_cascade` / `install_with_strategy` simplified accordingly.
- `import json`, `import zipfile`, `import tempfile` from
  `strategies.py` (only used by the dropped helpers).

## Tightened

- `DropZone.dragEnterEvent` now accepts only `.apk` (was
  `(".apk", ".xapk")`).
- `_on_file_dropped` / `_choose_file` lost their `.xapk` branches.
- `build_context_from_path` rejects `.xapk` (and any other extension)
  with `ValueError` - defensive backstop so a direct caller can't sneak
  multi-split input past the UI filter.
- `_strategy_hdb_broker_install` / `_strategy_pm_disable_install`
  guards collapsed: the multi-APK and non-`.apk` branches share the
  same single-input check now.
- `INSTALL_PARSE_FAILED_INCONSISTENT_CERTIFICATES` hint reworded
  (no longer mentions XAPK splits).

## Tests

- `test_build_context_for_xapk_with_obb` removed.
- `test_build_context_rejects_xapk` added - `.xapk` raises `ValueError`.
- `_ctx` helper lost `obb=` parameter; `_make_xapk` helper deleted.
- `test_installer.py` mock updated to return a single ctx (was tuple).

294 tests pass.

Version: 0.20.0 → 0.21.0 (minor - user-visible behavior change:
`.xapk` files are no longer accepted at the picker / drop zone).
