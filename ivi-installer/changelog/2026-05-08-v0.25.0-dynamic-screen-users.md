# 0.25.0 - Dynamic multimedia-screen user resolution

## The bug

Field testing on a non-Deepal-S09 head unit revealed that the active
driver user can land on **id 10 or 12** - sometimes with no user 13 at
all, and sometimes with extra users above 13. Both install strategies
(HDB-broker and pm-disable) plus several Tools/UI paths were pinned to
the canonical Deepal set `(10, 11, 12, 13)`, and to the magic number
`13` as the "driver" id, leading to:

* HDB-broker fan-out running `pm install-existing --user 13` on cars
  without a user 13 → driver screen silently missed.
* `_FALLBACK_CATEGORIES` returning `Driver=(13,), Passenger=(10,12),
  Rear=(11,)` on cars that don't have user 13 - a non-existent id sat
  under the Driver checkbox.
* The user-name heuristic forcing every unrecognised id into `rear`
  unless it equalled `13`.
* `_force_uninstall`, `GrantRuntimePermsWorker`, `_enroll_celia`, and
  `RecentPackagesWorker` iterating only over `(10,11,12,13)`, so any
  extra user (e.g. 14) was silently skipped.

The pm-disable strategy itself already used the right pattern via
`_resolve_packageinstaller_user` (= `max(running non-zero)`, mirroring
the competitor's `car_user_id`); the fix below extends that pattern
everywhere else.

## Fix

New `strategies._live_screen_users(serial)` helper: probes
`pm list users` and returns every running user with id ≥ 10, sorted
ascending. Falls back to `HDB_SCREEN_USERS = (10,11,12,13)` only when
the probe yielded nothing usable, so the cascade still has *some*
targets to try when the device is uncooperative.

Call sites that used `HDB_SCREEN_USERS` at runtime now go through this
helper:

* `strategies._resolve_targets` default branch (the "all screens" path
  used when `target_users` / `target_user` are unset).
* `strategies._force_uninstall`.
* `ui.workers.GrantRuntimePermsWorker`.
* `ui.main_window._enroll_celia`.
* `ui.workers.RecentPackagesWorker` (consolidated with the helper -
  was already half-dynamic but kept its own fallback list).

`devices.categorize_screens`:

* `multimedia` filter dropped the `<= 13` upper bound - users above 13
  now go through the same path.
* The `if u.user_id == 13` magic number in the user-name heuristic
  branch was replaced with `u.user_id == top_id`, where `top_id` is
  the highest running multimedia id. This matches the competitor's
  `car_user_id` and `_resolve_packageinstaller_user` semantics.
* The "all names opaque" branch now calls `_dynamic_fallback`, which
  builds `Driver=(top_id,)` and lumps every other id into `Rear`.
  Returns the historical `_FALLBACK_CATEGORIES` only when the live id
  set is `⊆ {10,11,12,13}` *and* contains 13 - preserves the known-
  good Deepal S09 layout while no longer misleading non-S09 cars.
* `_FALLBACK_CATEGORIES` (the constant) is now reserved for the
  "no multimedia users at all" case.

## Why this is more robust

* Display-based mapping (added in 0.23.2) remains the primary source -
  it's the most reliable signal we have. This patch only fixes the
  fallback paths that fire when display probes can't tell us anything.
* The HDB-broker strategy's stage 3 fan-out now lands on the user ids
  that actually exist on the device. No more `pm install-existing
  --user 13` against a phantom user; no more user 14 left behind.
* Behaviour stays consistent with the pm-disable strategy, which
  already used `max(running non-zero)` for its disable target.
* The "Install on" UI grid no longer points the Driver checkbox at a
  non-existent user when display probing falls through and names are
  opaque.

## Files

* `ivi_installer/strategies.py` - `_live_screen_users` helper, updated
  `_resolve_targets`, `_force_uninstall`.
* `ivi_installer/devices.py` - `_dynamic_fallback`, removed `<= 13`
  filter, replaced `== 13` magic numbers with `== top_id`.
* `ivi_installer/ui/workers.py` - `GrantRuntimePermsWorker` and
  `RecentPackagesWorker` use `_live_screen_users`.
* `ivi_installer/ui/main_window.py` - `_enroll_celia` uses
  `_live_screen_users`.
* `tests/test_strategies.py` - 4 new cases covering
  `_live_screen_users` filtering / fallback and `_resolve_targets`
  default-branch behaviour without and with explicit `target_users`.
* `tests/test_devices.py` - 3 new cases covering dynamic fallback
  when no user 13 exists, with a user above 13, and the heuristic
  path picking `top_id` instead of the magic 13.
* `ivi_installer/__init__.py`, `pyproject.toml` - version 0.25.0.
