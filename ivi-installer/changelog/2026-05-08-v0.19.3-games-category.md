# 0.19.3 - Games category added

The "Games" half of AppGallery's "Categories" tab tree (parent URI
`6c405fed6a3d432bbbfdd4541ea813ba`) is now pulled alongside Apps.
The parent URI returns a flat paginated mix from all six game
sub-genres (Sports / Action / Card & Board / Puzzle / Role-playing /
Strategy) - saturating naturally at ~120 items, well below the
`DEFAULT_MAX_PAGES = 20` cap.

## Catalog size

| | v0.19.2 | v0.19.3 |
|--|--:|--:|
| raw items pulled | 1025 | 1145 |
| unique after dedup | 864 | **984** |
| `games` entries | - | 120 |

## Code

- `sources/appgallery_index.py` - added `("games",
  "6c405fed6a3d432bbbfdd4541ea813ba")` before `most_ranked` in
  `DEFAULT_CATEGORY_URIS`. Order matters: a game that's also in Most
  Ranked gets tagged with `games` (specific) instead of
  `most_ranked` (catch-all).
- `catalog.py` - `games` added to `CATEGORIES`.
- `ui/store_tab.py` - `games` added to `CATEGORY_DISPLAY` between
  `shopping` and `most_ranked`.

Tests: 299 passed (no test changes needed - synthetic fixtures
don't reference games).

Version: 0.19.2 → 0.19.3.
