# 0.16.1 - Drop the section chip row

The `All / New / Recently updated / Tested` chip row was redundant
once the Sort dropdown shipped. Sort already exposes "Recently added"
and "Recently updated" (orderings of the same data, which is what
users actually want to browse), and `Tested` was a meaningful filter
only for the 4 bundled extras - F-Droid entries are all
`tested=False`, so the chip filtered to a near-empty list.

Removed:
- The chip strip widget at the top of the Store tab.
- `_section`, `_section_chips`, `_select_section`,
  `_refresh_section_chip_styles` from `StoreTab`.
- The 3 UI tests that drove the chips
  (`test_tested_section_chip_excludes_untested`,
  `test_section_new_chip_filters_by_added_at`,
  `test_section_chip_styles_track_active_state`).

Kept (unchanged):
- `CatalogStore.query_ids(section=...)` and the SQL section logic.
  The query layer still understands `new` / `updated` / `tested`;
  no UI consumes them, but the filter is well-tested and a future
  page (e.g. a "what's new" widget on the Install tab) can opt in
  without touching the SQL again.
- `catalog.SECTIONS`, `catalog.SECTION_RECENCY_DAYS`, the
  `search(section=...)` helper.

Version: 0.16.0 → 0.16.1.
