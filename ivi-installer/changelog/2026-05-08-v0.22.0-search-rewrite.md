# 0.22.0 - Search rewrite + drop the Category dropdown

The Store tab's Category filter never matched how a user actually
searches an app store: nobody picks "photo & video" first and then
scrolls. AppGallery's own web app puts a single search bar up front
with hot-keyword suggestions on focus and live type-ahead per
keystroke. v0.22 rebuilds the search experience around those two
endpoints and removes the Category selector entirely.

## What changed

### Catalog fetch
- `fetch_index` now walks **only** the Most Ranked URI
  (`automore|doublecolumncard|903219|A02000`). Cold-fetch goes from
  ~33 s Ă— 12 categories to ~3-5 s Ă— 1 list.
- `DEFAULT_CATEGORY_URIS` collapsed to a single tuple; new module-
  level constants `MOST_RANKED_LABEL` and `MOST_RANKED_URI` are the
  source of truth.
- `from_appgallery_index` no longer "snaps" the section label to a
  fixed taxonomy - whatever URI label was used surfaces verbatim on
  the entry. The bundled list always tags its entries `most_ranked`.

### Search dropdown
- New helpers in `sources/appgallery_index.py`:
  - `get_hot_search_list()` - POST `/edge/index/getnewhotsearchlist`
    and return a flat de-duped list of "popular searches". No JWT
    needed; the endpoint is unauthenticated JSON-in/JSON-out.
  - `complete_search_word(keyword)` - POST `/edge/index/completeSearchWord`
    and return `{keyword, suggestions, top_app}`. AppGallery's #1
    match comes back as a raw item dict so we can build a synthetic
    `CatalogEntry` and run it through the existing install pipeline.
- New workers `HotSearchKeywordsWorker` + `CompleteSearchWordWorker`
  drive both endpoints off the UI thread.
- `StoreTab` grew a focus-aware suggestion popup (`_SearchSuggestPopup`,
  parented to the tab so it doesn't grab the mouse like Qt.Popup
  does). Three new signals - `hot_keywords_requested`,
  `search_suggestions_requested`, `suggested_app_install_requested` -
  let main_window own the network calls and CatalogEntry assembly.
- Type-ahead is debounced via a 200 ms `QTimer.singleShot`. Each
  keystroke still updates the local SQLite filter (cheap LIKE), but
  only the LAST keystroke in a burst triggers the network call.

### Backend cleanup
- `CatalogStore.query_ids` and `count` lost the `category` parameter.
  The `category` column stays in SQLite (the row UI still prints it
  as "v1.2 Â· most_ranked") but the index `apps_category_idx` is
  dropped - nothing queries on it any more.
- `catalog.CATEGORIES` removed. `catalog.search()` keeps its
  `category` keyword-arg as a pure-function utility (some tests use
  it) but it's no longer exercised by the UI.

## Probe notes

The two `/edge/index/*` endpoints return 200 OK with no `Interface-
Code`, no cookies, no JWT bootstrap. Confirmed via `curl` against the
EU shard - `web-dre.hispace.dbankcloud.com`. They expect JSON in the
body (not form-urlencoded like `/edge/uowap/index`).
