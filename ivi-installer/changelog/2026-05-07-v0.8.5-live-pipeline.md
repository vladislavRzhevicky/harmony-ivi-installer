# v0.8.5 - install pipeline goes live + per-strategy stages

The pipeline panel and grant matrix on the Install tab were a visual
stub since the redesign. They now react to real backend progress.

## What changed

- **Per-stage events**: `InstallContext` gained a `stage_callback` and
  a `StageEvent` payload (`strategy`, `index`, `kind`, `hint`,
  `duration_ms`, optional `data`). Both install strategies emit these
  via the new `_StageReporter` helper at the boundary of each stage.
- **Strategy-shaped pipeline**: each strategy now declares its own
  `stages` tuple on `StrategyDescriptor`. The pipeline rebuilds when
  the user flips the strategy segment, and again at install time -
  `pm_disable_install` shows 6 rows (stage / adjust policy / install
  / restore policy / mirror / grant), `hdb_broker_install` shows 5
  (verify helper / stage / authorize / mirror / grant).
- **Live timing**: each stage carries `duration_ms`, formatted as
  "180ms" / "1.2s" / "2m 14s" on the right-hand column of the row.
- **Generic copy only**: hint snippets ("Staged", "Authorized",
  "Mirrored to 4 screens") never reveal commands, paths, package
  names, or any other reproducible details. The full debug log still
  lives in the log pane as before.
- **Grant matrix from real data**: replaces the synthetic
  `fill_demo_success` paint with `fill_from_summary(summary, users)`.
  Per the simplified spec, the matrix paints at column granularity -
  whole user column green when all permission grants succeeded for
  that user, red on any failure, em-dash for users not included in
  the run.

## Wiring

- `installer.install_cascade(..., stage_callback=…)` and
  `install_with_strategy(..., stage_callback=…)` accept the new
  callback (default no-op for headless callers).
- `InstallWorker.stage = Signal(object)` re-emits backend events on
  the Qt thread; `MainWindow._on_install_stage` updates the pipeline
  + matrix.
- `MainWindow._refresh_pipeline_stages()` is wired to both strategy
  radios - switching strategies repaints the pipeline rows.

## Compatibility

- No public API removed. `Pipeline.set_step` /
  `mark_done_with_timing` are gone (they were ad-hoc visual hacks
  with no backend signal behind them); the new callers go through
  `mark_running` / `mark_done` / `mark_failed` / `mark_skipped`.
- `INSTALL_STAGES` constant removed from `main_window.py` - the stage
  list now lives on each strategy descriptor.
