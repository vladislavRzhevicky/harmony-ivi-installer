# 0.18.0 - Live AppGallery integration + curated 40-app seed

Two related pieces shipped together:

## 1. extras.json grew from 4 to 40 RU/CIS apps

Replaces the placeholder list of 4 entries with a curated set of
apps useful for in-car head units in Russia / CIS - Yandex
(Maps/Navigator/Music/Browser/Go/Refuel/Metro/Weather/Disk/Search),
ВКонтакте (VK / Messenger / Calls / Video / OK), streaming
(Кинопоиск / START / MEGOGO / Plex), automotive (Дром, Авто.ру,
ГИБДД, OBDeleven, Radarbot), aux (OZON, Wildberries, OsmAnd,
Sygic, Yandex Browser/Старт, Castbox, ЛитРес, MyBook, Deezer, etc.)
+ Telegram via direct download.

The list is generated by **`scripts/build_extras.py`** - a one-shot
tool that:
1. Searches AppGallery for each curated entry by keyword
2. Picks the result whose `package` matches the expected one
3. Validates the C-id by probing `/appdl/<id>` - the redirect URL
   leaks the package name, catching stale/misallocated C-ids (e.g.
   we found that the previous extras's `ru.yandex.music` C-id now
   redirects to TikTok)
4. Emits `extras.json` with the verified `appid`/`name`/`size`/
   `version`/`icon_url`/`description_ru`

Re-run the script to refresh prices/versions; output is
deterministic given the same curated input.

## 2. AppGallery as a live catalog source (toggle in the UI)

A second toolbar pill - **"AppGallery"** - sits next to **F-Droid**.
Independent on/off. Toggled on, it pulls a configured set of
AppGallery category URIs (Top RU, Cars, Travel, Navigation) live,
caches the JSON for 24 h, and merges the results into the catalog
SQLite. ~160 unique apps added on top of extras.

Combined with the existing F-Droid toggle, the user picks any
combination of:
- extras only (default)
- extras + F-Droid (~4400 apps)
- extras + AppGallery (~190 apps)
- extras + F-Droid + AppGallery (~4500 apps)

Whichever toggles are active, the catalog SQLite is rebuilt from
the union; `extras.json` always wins on id collisions so curated
descriptions / `tested` flags survive the merge.

## How AppGallery API access works

The web app (`appgallery.huawei.com`) speaks to a JSON edge service
at `web-dre.hispace.dbankcloud.com/edge`. Discovered flow:

1. GET `/edge/webedge/getInterfaceCode` → JWT (~15 min lifetime,
   HMAC-bound to the User-Agent)
2. POST `/edge/uowap/index` form-urlencoded with header
   `Interface-Code: <jwt>_<ts_ms>`. Methods:
   - `internal.getTabDetail` - paginated category list
   - `internal.completeSearchWord` - autocomplete
3. Search via `getTabDetail` with `uri=searchapp|<keyword>`

All implemented in new `sources/appgallery_index.py` mirroring the
shape of `sources/fdroid.py` (cached `fetch_index`, threaded worker
in the UI flow).

## Code

- `sources/appgallery_index.py` - new. AppGallerySession,
  category_items, search, fetch_index.
- `catalog.py` - `from_appgallery_index` converter, `_APPGALLERY_KIND_MAP`
  Russian-tag-to-our-taxonomy mapping.
- `ui/workers.py` - `CatalogFetchWorker` accepts `include_fdroid` /
  `include_appgallery` flags, fetches each enabled source, merges,
  writes one SQLite.
- `ui/store_tab.py` - second source pill, refactored
  `_make_source_toggle` / `_apply_source_toggle_style` so both
  toggles share styling.
- `ui/main_window.py` - separate QSettings keys
  `store/fdroid_enabled` and `store/appgallery_enabled`,
  `_react_to_source_toggle` triggers a rebuild via the existing
  worker.
- `scripts/build_extras.py` - new. The ~/curated/ list is hand-edited
  here.
- `resources/extras.json` - re-generated, 40 verified entries.
