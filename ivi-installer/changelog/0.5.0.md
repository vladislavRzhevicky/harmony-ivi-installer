# 0.5.0 - 2026-05-06

The session that turned the IVI Installer from a multi-strategy
"shotgun" tool that didn't actually install on HarmonySpace 5.0 into a
focused single-path installer that does. Reverse-engineered the
leftover bypass agent on the car, identified the one working path,
trimmed everything else, and added runtime-permission grant via
reflection.

## Headline

Plain `adb install` and the existing strategies (`pm install`,
`cmd package install-create`, system-installer-intent, disable
verifier, bypass unknown-sources, etc.) all fail on HarmonySpace 5.0
(Deepal S09 / HwSAPT, ICHU 200.221.170, Android 12 SDK 31) with
`INSTALL_FAILED_ABORTED: User rejected permissions`. Huawei's
`HwInstallPolicy` hook intercepts every standard installer entry-point.

The car already had a leftover bypass agent running from a prior
workshop session - `app_process64 AvatrHdbBroker 38787`, listening on
`127.0.0.1:38787`. That agent reflects into Huawei's hidden
`IHwPackageManager` API and creates a `PackageInstaller.Session` with
the HDB flag (`installFlags |= 262144`) plus a SHA-256-signed
`hdbEncode` and `hdbArgs`, which Huawei's policy hook explicitly
trusts. We reverse-engineered the broker, discovered that
`pm grant <pkg> <perm>` is similarly intercepted but the underlying
`IPackageManager.grantRuntimePermission()` binder is NOT, and shipped
both as a single working install pipeline.

## Added

- **`_strategy_hdb_broker_install`** - the only install strategy that
  works on HarmonySpace 5.0. Four-stage pipeline:
  1. `adb forward tcp:38787` + PING the on-device broker.
  2. Push APK to `/data/local/tmp/iviinstaller_<pkg>.apk`.
  3. `RUN hdb-session-install-user - <apk> <user> <nonce>
     com.huawei.appinstaller.car 1000` over the broker's TCP wire
     protocol - creates an HDB-flagged install session that
     `HwInstallPolicy` lets through.
  4. Fan out to every multimedia-screen user (10/11/12/13) via the
     stock `pm install-existing --user N` (no broker needed once the
     APK has been validated globally).
  5. Grant every requested-but-ungranted runtime permission per user
     via the bundled helper jar (see below).
- **Runtime-permission grant helper** - a 2.5 KB Java helper
  (`ivi_installer/resources/HwPermGrant.java` → `hw-perm-grant.jar`,
  built by `tools/hw-perm-grant/build.sh`) that calls
  `IPackageManager.grantRuntimePermission(pkg, perm, userId)` directly
  via reflection from `app_process`. This bypasses Huawei's pm-shell
  hook (which silently drops `pm grant` calls). Confirmed-working on:
  POST_NOTIFICATIONS, ACCESS_FINE/COARSE_LOCATION, RECORD_AUDIO,
  CAMERA, READ_PHONE_STATE, READ/WRITE_EXTERNAL_STORAGE,
  READ_CONTACTS, GET_ACCOUNTS, BLUETOOTH_CONNECT.
- **Tools tab** in the GUI:
  - "Run diagnose" - read-only system probe (services, settings,
    appops, getprop, install handlers, forensics) that drops a report
    next to the app log.
  - "Grant runtime permissions" - combobox listing third-party
    packages on the device + a button that grants all ungranted
    runtime perms across users 0/10/11/12/13. Useful for packages that
    were installed before this version, or via another tool.
- **`ThirdPartyPackagesWorker`** + **`GrantRuntimePermsWorker`** -
  Qt workers backing the Tools tab without blocking the UI thread.
- **`HUAWEI_HDB_BYPASS_RECIPE.md`** - full technical recipe at the
  repo root: how the bypass works, broker wire protocol, command
  reference, the 8 gotchas that wasted a session, and a Python helper
  script.
- **`NEXT_SESSION.md`** - handoff doc for picking up cold: state of
  v0.5.0, queued features (broker resilience, force reinstall, IME
  enrolment) with locked-in design decisions, build cheat-sheet.
- **`README.md`** at repo root - one-page overview pointing at the
  recipe + handoff.
- **Memory note** `project_deepal_hdb_bypass.md` - single-screen
  pointer for the next Claude session.

## Changed

- **Strategy registry trimmed** from 13 entries to 3:
  `hdb_broker_install` (the only working install path), `diagnose`
  (read-only), `app_process_helper` (stub for future broker redeploy).
  Old strategy functions are still in `strategies.py` (legacy tests
  reference them) but no longer surface in the UI.
- **Install tab simplified**:
  - Removed nested per-strategy tabs (`Auto`, `pm install`, etc.).
  - Removed the `Target user` combobox, `Installer pkg` text input,
    and `Auto-grant runtime permissions` checkbox.
  - Replaced with a single "Install" button + a 2-radio target
    chooser: "All multimedia screens" (default - seed via user 0,
    fan-out to 10/11/12/13) or "Driver only (user 13)".
- **File picker accepts `.apk` only**. Selecting a `.xapk` shows a
  warning explaining that the multi-APK bridge command hardcodes the
  Avatr installer package and won't work on Deepal - recommends
  downloading a universal APK instead.
- **Default installer package switched** from `com.huawei.appmarket
  .vehicle` (the Avatr default hardcoded in `HuaweiShellBridge`) to
  `com.huawei.appinstaller.car`. The Avatr one doesn't exist on
  Deepal, so passing it produces `INSTALL_FAILED_INTERNAL_ERROR`. The
  switch is critical and is the single most important fix in this
  release.
- **Diagnose strategy** now picks its output directory from a chain
  (`~/Desktop` → `~/Documents` → `~/.ivi-installer/diagnose` →
  `mkdtemp`) with a write-probe per candidate, then prints the chosen
  path up-front. Falls back gracefully if Desktop is unwritable.
- **`pyproject.toml`** uses
  `[tool.hatch.build.targets.wheel.force-include]` so the helper jars
  ship in the bundle.
- **App version** bumped through 0.1.0 → 0.5.0.

## Fixed

- **Diagnose `UnboundLocalError`** - `desktop_pulls = os.path.join
  (desktop, ...)` referenced `desktop` before it was assigned at the
  bottom of the function. Moved the directory-resolution block to
  the top, so subsections that pull APKs always have somewhere
  to put them.
- **Stage-4 grant `NameError: name 'os' is not defined`** - a
  late-night fix attempt referenced bare `os.unlink` in
  `_ensure_perm_helper_pushed`'s `finally`. Renamed the local import
  to `_os` and used it consistently.
- **Tools-tab "Refresh packages" `TypeError: sequence item 1:
  expected str instance, list found`** - `ThirdPartyPackagesWorker`
  was passing a list to `adb.run` whose signature is `*args: str`.
  Switched to the variadic call shape.
- **Bridge call returning 0 bytes** when client did
  `socket.shutdown(SHUT_WR)` - the broker's `readLine()` interpreted
  EOF on the read side as end-of-frame and closed the socket without
  writing a `RESULT` line. Removed the half-close on our side.

## Removed

- Strategy registry entries for paths that don't work on this
  firmware (kept the function bodies so legacy tests still compile,
  but they're invisible to users): `pm_install_streamed`,
  `pm_install_from_tmp`, `cmd_package_session`, `system_installer
  _intent`, `disable_verifier`, `bypass_unknown_sources`,
  `pm_install_with_auto_confirm`, `try_root_install`,
  `logcat_during_install`, `launch_hidden_installer`,
  `capture_install_dialog`.

## Known issues left for the next release

- **Broker resilience** - the bypass relies on a leftover
  `app_process64` process from 2026-04-28. If it dies (reboot,
  `/data/local/tmp` wipe), the app silently fails at the health
  check. Auto-redeploy from a bundled `avatr-hdb-broker.jar` is the
  first item queued for 0.6.0.
- **Downgrade installs** fail with
  `INSTALL_FAILED_VERSION_DOWNGRADE` (Celia Keyboard, KinoPoisk old
  builds). A "Force reinstall" checkbox that does
  `pm uninstall --user N` first is queued.
- **IME enrolment** - keyboards (Celia, Yandex) install but never
  become the active input method. HarmonySpace hides the IME picker
  in user-facing Settings. The plan: extend the helper jar with
  `ime-enable` + `ime-set-default` subcommands that write
  `Settings.Secure.{enabled_input_methods,default_input_method}`
  via reflection (same pattern as the grant fix).
- **`com.huawei.appinstaller.car` is hardcoded** for Deepal. Avatr
  cars use `com.huawei.appmarket.vehicle`; runtime auto-detection of
  which exists is a 5-line change but needs an Avatr car to test on.
- **User-11/13 install rollback** - on subsequent HDB installs of
  other packages we've seen older packages drop back to
  `installed=false` for users 11 and 13. Cause unclear. Workaround:
  re-run install. Investigation deferred.
- **Windows packaging** - scaffolded in `pyproject.toml`, never
  built. Needs a Windows agent.
