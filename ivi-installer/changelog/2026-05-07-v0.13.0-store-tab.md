# 0.13.0 - In-app Store catalog (F-Droid + AppGallery + Direct)

A new **Store** tab (between *Install APK* and *Tools*) lets the user
browse a curated list of apps known to work on the head unit and click
*Install* - discovery + download + install run end-to-end through the
existing pipeline. No new install path is introduced; the Store is
discovery-only and hands the resolved APK to `installer.install_cascade`.

## Architecture

- New package `ivi_installer/sources/` - one module per `kind`,
  every module exposes `resolve(entry, *, out_dir, progress,
  log_callback) -> Path`. Ships:
  - `sources/fdroid.py` - full discovery via the public
    `https://f-droid.org/repo/index-v2.json`. Disk-cached for 24 h with
    stale-cache fallback on network errors. Picks the highest
    `versionCode` per package, validates payload (size + ZIP magic +
    sha256 from the index).
  - `sources/appgallery.py` - moved from `ivi_installer/appgallery.py`
    (one-line import shim was avoided; all callers updated). Adds a
    thin `resolve()` wrapper that maps a catalog dict → existing
    `download(app_id, …)`.
  - `sources/direct.py` - trivial URL streamer with the same payload
    guards plus optional sha256 verification.
- `ivi_installer/sources/__init__.py` - registry `RESOLVERS` (dispatch
  by `kind`). Adding a future source = drop a module + register here.
  No UI or schema changes needed.
- `ivi_installer/catalog.py` - `CatalogEntry` dataclass + `load()` /
  `search()`. Bundled-only in 0.13 (no remote fetch yet).
- `ivi_installer/resources/catalog.json` - 9 curated entries: NewPipe,
  Organic Maps, AntennaPod, VLC, Maps.me (F-Droid); Радиопоток,
  Кинопоиск, Яндекс Музыка (AppGallery); Telegram (direct).
- `ivi_installer/ui/store_tab.py` - the tab widget. Toolbar +
  scrolling `AppRow` list + footer + loading/empty/error states, all
  driven from the design tokens already in `theme.py`.
- `StoreInstallWorker` in `ui/workers.py` - chains `sources.resolve` →
  `installer.install_cascade` and emits per-phase signals (`resolving`
  / `downloading` / `installing` / `success` / `failed`) so the row's
  Install button mirrors the design's per-phase states.

## Icons

Each catalog entry carries an `icon_url`. The Store tab lazy-loads
icons via a shared `QNetworkAccessManager` (`IconLoader` in
`store_tab.py`), with a per-URL on-disk cache under
`<QStandardPaths.CacheLocation>/ivi-installer/store-icons/`. Until a
real icon arrives the row falls back to a two-letter monospace plate
(matches the design handoff).

## Tests

- `test_catalog.py` - bundled-catalog smoke + every search filter +
  bad-row skip.
- `test_sources_fdroid.py` - version picker, URL builder, sha256
  verifier, in-memory index fixture, cache TTL + stale fallback.
- `test_sources_direct.py` - happy path + HTML / tiny / non-ZIP /
  sha256 mismatch rejection.
- `test_sources_registry.py` - dispatch routes + error paths.
- `test_store_tab.py` - pytest-qt build, every filter, install signal,
  per-row phase rendering.
- `test_sources_appgallery.py` - moved from `test_appgallery.py`.

## Out of scope (deferred to later versions)

- Remote catalog fetch (`v0.14`).
- APKPure / RuStore sources (the `sources/` plug-in interface is ready).
- Auto-update detection for already-installed apps.
