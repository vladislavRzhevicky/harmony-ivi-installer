# 0.14.2 - Store rows no longer pop out as separate windows on filter

A search/filter change in the Store tab made every previously-rendered
row turn into its own top-level macOS window - complete with traffic-
light titlebar - instead of disappearing.

Cause: `_detach_rendered()` called `setParent(None)` on each row
widget. For a visible Qt widget, reparenting to None promotes it to a
top-level window. The next `_render_more()` reparented some of them
back into the layout, but rows that didn't make it into the new
filter were already detached and visible as standalone windows.

Fix: detach by `hide()` only - the widget stays parented to the
scroll-area's inner widget, just outside the layout. `addWidget()` in
`_render_more()` re-attaches and `setVisible(True)` shows it again.
No `setParent(None)` on the live-cache path; the discard path
(`_discard_cached_rows`, used by `set_catalog`) still detaches +
`deleteLater` because those widgets are about to die.

Version: 0.14.1 → 0.14.2.
