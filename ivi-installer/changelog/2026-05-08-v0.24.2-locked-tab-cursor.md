# 0.24.2 - Forbidden cursor on locked tabs

Hovering the locked Tools / Enable ADB tabs now switches the cursor
to `Qt.ForbiddenCursor`, so the lock is visible before the user
clicks.

## What changed

- macOS segmented bar (`MacSegmentedTabBar`) no longer calls
  `setEnabled(False)` on a locked button - Qt suppresses hover
  events on disabled widgets, which means the `setCursor` we'd
  set was never displayed. The button stays enabled, click+keyboard
  switching is gated by an internal `_locked` list, and a
  `[locked="true"]` QSS property paints it as muted.
- A `_ForbiddenCursorOnDisabledTab` event filter sits on the
  native `QTabBar` (Win/Linux) and recomputes the cursor on each
  `MouseMove` based on `tabAt(pos)` + `isTabEnabled(idx)`. Mouse
  tracking is enabled on the bar so the filter fires without a
  pressed button.
