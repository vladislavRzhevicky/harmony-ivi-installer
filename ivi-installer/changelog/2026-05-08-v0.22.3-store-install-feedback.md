# 0.22.3 - Store install: visible feedback + fast-fail race fix

The Store tab's Install button looked "dead" on click - the user got
no toast, no progress strip, nothing they could rely on to know the
install had even started. A second click then surfaced "Wait for the
current operation to finish" because `_busy` had silently flipped to
True. Two distinct issues feeding the same symptom:

## 1. Race in `worker.finished` connection

`_on_store_install` connected the `_on_op_finished` lambda AFTER
calling `workers.run_in_thread(worker)`. `run_in_thread` calls
`thread.start()` before returning, so a worker that fast-fails
inside `run()` (e.g. raises before any phase emits, or hits a
problem during the very first source resolution) emits its
`finished` signal BEFORE the main thread reaches the `connect()`
call. The lambda is then never registered and `_on_op_finished`
never runs - `_busy` stays True for the lifetime of the process.

Fix: inline the QThread setup in `_on_store_install` and wire every
`worker.finished` receiver BEFORE `thread.start()`. Same pattern as
the search-suggestion workers (which already had the same issue in
v0.22.0 and were patched in v0.22.1 with the in-flight-list approach).

## 2. No up-front toast

When the install IS rolling, the row's button transitions to
"Resolving…" → "Downloading NN%" → "Installing…". But if the row
isn't visible (off-screen, filtered out, or the install was kicked
from the search-suggestion popup against a synthetic CatalogEntry
whose package isn't in the active catalog at all), the user sees
nothing.

Fix: emit a confirmation toast - `Installing <name>…` - at the top
of `_on_store_install`. The phase-update path stays unchanged for
visible rows; the toast is the safety net.

## 3. Bonus: log dropped phase updates

`StoreTab.set_install_phase` now logs (debug-level) when
`_ensure_row` returns None for a phase event. Helps trace search-
suggestion installs that don't have a matching catalog row, without
spamming the in-app log panel.

## Tests

309 still pass - no behaviour change visible to existing tests, the
fixes are around timing (race) and side-channel UX (toast).

Version: 0.22.2 → 0.22.3 (patch - bug fix for an install path that
looked silent and could leave the app stuck busy).
