# 0.24.1 - Lock Tools and Enable ADB tabs

The Tools tab moved to the second-to-last position (right before
Enable ADB) and is now disabled, alongside Enable ADB. Both tabs
remain in the layout but are unclickable in both the macOS
segmented control and the native QTabBar - they expose internal
device controls that shouldn't be reachable from the regular UI.

## What changed

- New tab order: Install APK, Store, Timezone, Device info,
  Keyboards, Tools, Enable ADB.
- `QTabWidget.setTabEnabled(5, False)` and
  `setTabEnabled(6, False)` on Tools and Enable ADB.
- `MacSegmentedTabBar` gained `setTabEnabled(idx, enabled)` so
  the macOS pill-bar respects the same lock; locked items
  show greyed-out text and a `ForbiddenCursor`, and clicking
  them does not switch the page.
