# 0.10.0 - bundle adb inside the macOS .app

## Why

Symmetrical to the v0.9.0 Windows change: drop the «download
platform-tools on first launch» step on macOS too. Removes the firewall
/ no-internet / dismissed-dialog failure mode at customer sites and
gives every user a known, pinned `adb` build.

## What changed

Ship two darwin files inside the wheel, preserving the `lib64/` layout
that the platform-tools binary expects:

```
ivi_installer/resources/platform-tools/darwin/
  adb                      (20 MB, universal2 - Intel + Apple Silicon)
  lib64/libc++.dylib       ( 2.6 MB)
```

Pulled from `platform-tools-latest-darwin.zip` on `dl.google.com` -
same release as the Windows side (build 37.0.0-14910828).
`@executable_path/lib64/libc++.dylib` is how Google links adb on
macOS, so the lib64/ subdirectory must travel with the binary.

`adb._bundled_adb()` now also resolves the darwin path. On POSIX
hatchling reading source files off an NTFS checkout drops the
executable bit - so `_bundled_adb()` re-adds `+x` idempotently on
first lookup, swallowing OSError on read-only volumes (logged at
WARNING).

`pyproject.toml` adds the two darwin files to the wheel
`force-include` map and refreshes the «no bundled adb» comment in the
macOS briefcase section.

## What this does NOT do

- Does not bundle `adb` for Linux. Linux still relies on `ensure_adb()`
  to download platform-tools at runtime. Easy to add later if/when
  someone ships a `.deb`/`.rpm`.
- Does not add codesign / notarization. `briefcase package macOS
  --adhoc-sign` still applies; an ad-hoc signed `adb` runs fine on
  Apple Silicon, but full distribution outside dealer machines would
  need a Developer ID + notarization pass over the bundled binaries.

## .app size impact

~+22 MB to the unpacked `.app` Resources/. The `.dmg` compresses
this further (libc++ is highly compressible).

## Files touched

- `ivi_installer/__init__.py` - 0.9.0 → 0.10.0
- `pyproject.toml` - version, force-include the two darwin files,
  refresh the macOS bundle comment
- `ivi_installer/adb.py` - `_bundled_adb()` extended to darwin, hoisted
  `import sys` to module top
- `ivi_installer/resources/platform-tools/darwin/{adb,lib64/libc++.dylib}`
