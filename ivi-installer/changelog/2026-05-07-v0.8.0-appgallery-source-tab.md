# v0.8.0 - AppGallery URL surface + competitor parity in `appgallery.py`

## What

The Install tab now has a segmented **File / From URL** picker over a
shared source-card (matches `design/exports/searching-no-device-
handoff/searching-screen.html` - `.source-card` + `.source-tabs`).
The "From URL" tab exposes the existing AppGallery downloader that
was previously only reachable through a hidden `Ctrl+Shift+D` dialog.

The hotkey survives but now jumps to the URL tab and focuses the
input instead of opening a modal `QInputDialog`.

## Network behavior - aligned with AvatrAppInstaller

We compared `ivi_installer/appgallery.py` against the competitor's
`_do_gallery_install` (decompiled at `AVATR/decompiled/_tmp_partner.py`
+ `AVATR/analysis/tmp_partner_disasm.txt`) and adopted its choices on
points where the two diverged:

| Behavior              | Was (ours)                           | Now (competitor parity) |
|-----------------------|--------------------------------------|-------------------------|
| `User-Agent`          | `Mozilla/5.0 (Linux; Android 12…)`   | `Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36` |
| `Accept` header       | `application/vnd.android.package-archive,*/*;q=0.8` | (omitted) |
| SSL verification      | default (verify hostname + cert)     | `check_hostname=False`, `verify_mode=CERT_NONE` |
| Timeout               | 60 s                                 | 120 s |
| Tiny-payload guard    | (none)                               | reject downloads `< 10 000 bytes` |

Why disable cert verification: Huawei's CDN sometimes serves through
hosts whose chain is unreachable from desktop trust stores, and the
download silently fails on otherwise-fine systems. The competitor
ships with verification off for this reason. We still validate the
payload itself (HTML response → reject, tiny response → reject,
non-`PK` magic → reject), so an attacker-on-path can't silently
substitute a non-APK; the integrity check is downstream.

## What we kept that the competitor doesn't have

These aren't conflicts - they're additive safety / UX:

- `Content-Disposition` filename parsing (so downloaded files land
  with the actual app name instead of `C12345.apk`)
- `_safe_filename()` to strip path traversal and reserved chars
- ZIP magic check (`PK\x03\x04`) layered on top of the tiny-payload
  guard
- Atomic `.part` → final rename, so a partial download can't
  overwrite a previously good file

## UI wiring

- `main_window.py` - old hidden `appgallery_*` widgets replaced by a
  visible source-card containing a `QStackedWidget` over `File`
  (drop zone + APK card) and `From URL` (input + Download button +
  progress bar) pages.
- The Download button is gated on `appgallery.parse_app_id(...)`
  succeeding on the input - so the user gets immediate feedback that
  what they pasted will work, without hitting the network.
- `_prompt_appgallery` (Ctrl+Shift+D) now switches to the URL tab
  and focuses the input rather than opening a `QInputDialog`.
