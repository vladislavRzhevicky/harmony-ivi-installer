# 2026-05-06 - v0.7.0: competitor parity + dead-code cleanup

Reverse-engineered the AvatrAppInstaller competitor (`AVATR/` -
PyInstaller + Tkinter, decompiled with pycdc/pycdas) and pulled in the
parts that improve our UX or simplify the install flow. Also stripped
~1300 lines of dead code inherited from the v0.4 multi-strategy era.

Commit: `4ad14aa` on `0.7.0` (and `dev`).

## New: pm-disable install strategy (`strategies.py`)

`_strategy_pm_disable_install` - adopted from the competitor's primary
bypass:

```
pm disable-user --user 0 com.android.packageinstaller
pm install -r -d -g -i <huawei-installer> <remote-apk>
pm enable com.android.packageinstaller
```

Cheaper than HDB session install: no on-device broker, no
`app_process` helper, no jar push. Auto-detects the Huawei installer
pkg via `firmware.detect()` (Avatr → `appmarket.vehicle`,
Deepal → `appinstaller.car`) and falls back to the alt-installer if
the first one fails. Re-enables PackageInstaller in `finally` even on
mid-install crash.

Wired as the **fallback**; HDB broker remains the primary because
it's the proven HwSAPT 5.0 path. Order is selectable in the UI;
either choice cascades to the other on failure.

## New: firmware profile detection (`ivi_installer/firmware.py`)

Probes the device with `pm list packages` / `pm list users` /
`getprop ro.build.version.sdk` and returns a `FirmwareProfile` with
the right installer pkg, screen-user IDs, API level, and a label.
Falls back to `DEFAULT_PROFILE` (Deepal / API 31) on probe failure.
Used by `pm_disable_install` to avoid hardcoding the Deepal-specific
values, and ready for the Avatr-support branch.

## New: AppGallery direct downloader (`ivi_installer/appgallery.py`)

Paste a Huawei AppGallery link or `C12345`-style id into the install
tab. The downloader:

- Tolerantly parses any of `C101898721`, full URLs, or text
  containing the id token (`parse_app_id`).
- Streams from `https://appgallery.cloud.huawei.com/appdl/{id}` with
  a mobile UA, Content-Disposition aware filename, ZIP magic check,
  partial-file → atomic-rename.
- Drops the .apk into `~/Downloads/ivi-installer/` and pre-loads it
  into the picker so a subsequent click of Install just works.

Worker: `AppGalleryDownloadWorker` in `ui/workers.py`.

## New: Keyboards tab + Celia install + IME enrollment

Bundles `celia-keyboard-11-0-5-352.apk` (Russian + English Huawei
IME) into `ivi_installer/resources/`. Materializes to a tempfile on
first use so `adb push` can read it. Two buttons:

- **Install Celia Keyboard** - runs the same install pipeline (HDB
  primary → pm-disable fallback or vice versa), then auto-enrolls
  the IME.
- **Enable / set default only** - skips install when Celia is already
  on the device.

Enrollment uses the partner's discovery: `ime enable --user N <id>`
and (optionally) `ime set --user N <id>` shell commands directly,
with no reflection helper. Iterates user 0 + every multimedia
screen. Below the buttons: a live IME list (`ime list -s -a` parsed)
showing each registered IME with an enabled/disabled mark.

New helpers in `strategies.py`: `enable_input_method`,
`set_default_input_method`, `list_input_methods`, `discover_ime_id`.
New worker: `IMEEnableWorker`.

## Strategy radio + force-reinstall in Install tab

- **Strategy radio**: `pm-disable (lighter)` vs `HDB broker (5.0
  path)`. Selection is persisted to
  `~/.ivi-installer/settings.json` under
  `primary_install_strategy`. Default for fresh installs:
  `hdb_broker_install`.
- **Force reinstall** checkbox: triggers per-user `pm uninstall`
  before push. Recovers from `INSTALL_FAILED_VERSION_DOWNGRADE` (e.g.
  installing KinoPoisk 7.76 over the existing 7.82).
- Helper text: "If the chosen strategy fails, the other one runs as
  a fallback automatically." (No "cascade" wording - there are only
  two strategies.)

`InstallContext` now carries `force_reinstall: bool`. `installer.py`
threads it + `primary_strategy` to `run_cascade(ctx, primary=…)`.

## Removed: ~1300 lines of legacy install-strategy code

The pre-5.0 codebase shipped a 10-strategy cascade trying every angle
on the install dialog (streamed install, intent dispatch, root
install, verifier disable, auto-confirm via uiautomator dump, unknown-
sources bypass, …). All of that was dead code on the only firmware
we target. Removed:

- `_strategy_pm_install_streamed`
- `_strategy_pm_install_from_tmp` + `_install_from_tmp_session`
- `_strategy_cmd_package_session`
- `_strategy_system_installer_intent`
- `_strategy_disable_verifier`
- `_strategy_pm_install_with_auto_confirm` + `_find_install_button_coords`
- `_strategy_bypass_unknown_sources`
- `_strategy_logcat_during_install`
- `_strategy_try_root_install`
- `_strategy_launch_hidden_installer`
- `_strategy_capture_install_dialog`

Plus their support: `_VERIFIER_KEYS`, `_INSTALL_BUTTON_KEYWORDS`,
`_UNKNOWN_SOURCES_*`, `_SESSION_ID_RE`, `_build_install_flags`,
`_push_obb_best_effort`, `_abandon_session`.

`installer.py` rewritten from ~480 lines to ~110: dropped
`install_single`, `install_xapk`, `_install_split_multiple`,
`_install_split_session`, the legacy `install()` / `uninstall()`
dispatchers, `InstallResult`, the duplicated `_HINTS` table, and
`_parse_pm_install_output`. Surviving public API: `install_cascade`,
`install_with_strategy`, plus type/helper re-exports.

`strategies.py`: 3248 → 1928 lines.
`installer.py`: 481 → 110 lines.

## Open questions / not yet verified

- `pm_disable_install` works on Avatr 11 (the competitor relies on
  it) and very likely on HwSAPT 5.0, but real-car verification is
  pending. The fallback to HDB broker covers the worst case.
- `ime enable / set` works on Avatr 11 (the competitor relies on
  it). Whether HwSAPT 5.0 hooks the shell `ime` surface is unknown;
  the bundled fallback path (helper jar with reflection) is wired
  but not yet plumbed into the IME enrollment flow.
- `xcode.it.com` DoIP/UDS unlock - competitor ships it, we don't:
  our model assumes a dealer scanner unlocks the car first. No
  change vs v0.6.0.
