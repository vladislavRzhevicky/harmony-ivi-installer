# 0.11.2 - Fix passenger/rear screen mapping

In-cabin testing on the Deepal S09 showed the "Install on" checkboxes
had Passenger and Rear swapped: ticking **Rear** actually pushed the
APK to the front-passenger screen, and vice versa.

## Fix

`devices._FALLBACK_CATEGORIES` had the user-id assignments reversed:

- `passenger` was bound to user `11` (which is physically the rear
  seat-back screen on this car).
- `rear` was bound to users `10, 12` (the front passenger screen).

Swapped them back to the layout that matches the cabin:

- `passenger` → users `10, 12`
- `rear` → user `11`

Driver (`13`) is unchanged - that mapping has held on every Huawei IVI
we've tested.

## Files

- `ivi_installer/devices.py` - swap `user_ids` in `_FALLBACK_CATEGORIES`.
