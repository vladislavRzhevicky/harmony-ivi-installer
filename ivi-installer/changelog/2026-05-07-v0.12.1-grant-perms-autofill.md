# 0.12.1 - Auto-populate Grant runtime permissions package picker

The "Grant runtime permissions" card in the Tools tab had a picker that
required a click on **Refresh** before any installed packages appeared.
The placeholder asked for `com.example.app` - which nobody knows by
heart. The combo box was already a `QComboBox` with full machinery
behind it (`ThirdPartyPackagesWorker` runs `pm list packages -3`); we
just never auto-triggered it.

## Changes (`ivi_installer/ui/main_window.py`)

- `_on_tab_changed`: when the user switches to the Tools tab and the
  picker is empty, kick `_kick_pkg_list()` automatically. The combo
  shows installed third-party packages without the user having to know
  about Refresh.
- On serial change (different device connected): clear the combo so
  stale entries from the previous device don't linger. If the user is
  already on the Tools tab, refresh immediately.
- Updated placeholder text from
  `"e.g. com.example.app  (or click Refresh to list installed)"`
  to
  `"Pick an installed package, or type a name (Refresh re-fetches)"`.

The Refresh button still exists for cases where the device installed a
package while the picker was open (e.g. user just installed an APK and
wants to grant perms manually right after).

## Notes

- No backend changes. `pm list packages -3` already filters out system
  packages - the dropdown only shows third-party (user-installed) apps.
- "Installed by us" filtering (via installer attribution) was considered
  but skipped: factory-pre-installed apps share the same Huawei
  installer pkg, so the filter wouldn't reliably separate "installed
  via this tool" from "shipped on the IVI". Local install-history
  tracking would be a cleaner future iteration.
