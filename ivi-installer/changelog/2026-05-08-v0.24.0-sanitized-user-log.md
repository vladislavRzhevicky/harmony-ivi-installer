# 0.24.0 - Sanitized user log

The on-screen LOG pane now shows only short, human-readable phase
markers - never adb commands, device replies, strategy names, or
internal paths. The full verbose transcript still goes to the rotating
log file on disk and remains accessible via Copy log / Save log for
support.

## What changed

- Split `_log` into two helpers in `MainWindow`:
  - `_log_full(line)` - file only (rotating `~/.ivi-installer/logs/`).
    Every `worker.log_line` emission now lands here. This is where
    every adb command + device output continues to be recorded.
  - `_log_user(line)` - appears in the on-screen LOG pane (and is
    also tagged `[user]` in the file log so the support transcript
    has the same anchors the user saw, in order).
- Reclassified all `MainWindow._log` call sites: phase markers and
  outcomes (Installing X…, ✔ Install complete, ✘ Install failed,
  Connecting…, etc.) become user-visible; verbose breadcrumbs
  (strategy names, broker names, IP gateways, package lists with
  user IDs, traceback dumps) stay file-only.
- `Copy log` now copies the FULL on-disk log instead of the visible
  short summary, so support pastes still contain the complete
  diagnostic trace.

## Why

A regular user shouldn't be able to read off the install pipeline,
broker daemon name, exact adb commands, DoIP/TLS gateway IPs, or
device replies straight from the UI. The log file keeps everything
for triage; the on-screen pane is now intentionally opaque about
*how* the work happens.
