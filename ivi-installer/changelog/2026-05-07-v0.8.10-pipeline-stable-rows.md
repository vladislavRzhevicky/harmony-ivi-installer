# v0.8.10 - Pipeline row count stays stable

The "Mirror to passenger screens" row is no longer rendered in the
pipeline. The mirror code itself (the `pm install-existing` fan-out
to additional screens) still runs as part of every multi-screen
install - it's only the visible row that goes away. This kills the
layout jump that happened when toggling Simple/Complex strategies
or unchecking screen categories.

## What changed

- `StrategyDescriptor.stages` is now a `tuple[(code_index, label)]`
  instead of `tuple[label]`. The explicit code-indexes let the
  visible list skip arbitrary stages without desynchronising the
  remaining ones (grant-perms keeps its real code index even after
  mirror is dropped).
- Removed the `single_user_skips` field - its only use case (hide
  mirror when only one screen targeted) is now baked into the
  always-hidden treatment of mirror.
- `stages_for(...)` keeps its signature (still accepts `target_users`
  / `target_user`) for API compat but no longer filters; every
  strategy now exposes a fixed visible stage list.
- Strategy code is unchanged - `_StageReporter` still emits start /
  done / failed for the mirror stage; the UI just ignores those
  events because they aren't in the index map.
